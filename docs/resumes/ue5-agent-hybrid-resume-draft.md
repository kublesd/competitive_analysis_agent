### 竞品分析 Agent

项目链接：[填写 GitHub 或本地项目链接]  
技术栈：Python、LangGraph、LangChain、Pydantic、Streamlit、Tavily、pytest

项目描述：  
开发一个面向产品竞品分析的 AI Agent，将产品调研任务拆分为规划、检索、抽取、分析、验证和报告生成等阶段，最终输出带引用来源的 Markdown 竞品分析报告。

与游戏开发结合的表述角度：

- 该项目体现了将复杂任务拆成可编排工作流的能力，可迁移到游戏 AI 行为链、智能 NPC 决策链、开发者工具链和内容生成流程。
- Evidence、引用校验和 Verifier 机制可迁移到“AI 生成内容可追溯”“NPC 设定一致性检查”“剧情 / 任务生成质量检查”等游戏 AI 场景。
- Streamlit UI 和后台日志经验可迁移到策划工具、数据分析工具、内部编辑器和 AI 辅助生产工具。

核心工作：

- 使用 LangGraph 编排 Planner、Researcher、Extractor、Analyst、Verifier、Reporter 多阶段流程，支持验证失败后有限重试。
- 使用 Pydantic 定义 Agent 输入输出契约，确保模型结果可以被后续节点稳定消费。
- 接入 Tavily Search API，并封装 SearchProvider / SearchAdapter，统一处理搜索结果、URL 去重、官方来源分类和错误隔离。
- 实现引用验证与报告生成机制，保证最终结论可追溯到 Evidence，避免模型在报告阶段新增不可验证事实。
- 搭建 Streamlit 页面和后台日志，为每次分析生成 `analysis_id`，记录节点轨迹、耗时、证据数和异常类别。
- 建立固定评测集和 pytest 测试，覆盖成功、重试恢复和失败告警场景。

项目成果：

- 完成端到端 Agent MVP，可从用户输入自动生成带证据引用的竞品分析报告。
- 固定评测集案例通过率 100%，引用有效率 100%，来源覆盖率 100%。
- 形成“模型生成 + 代码约束 + 可观测日志 + 离线评测”的 Agent 工程实践。
